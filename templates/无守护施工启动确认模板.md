# 无守护施工启动确认

> 包工头识别到无守护触发词后第一时间产出。**时间契约 + 队列补给 + 临时分支推送策略**必须写清。
> 落盘位置：`docs/施工日志/<今天>/无守护启动-HHMM.md`
> 适用 playbook：`playbooks/16-无守护自动施工模式.md`

## 自动模式硬性护栏

- 默认 S / M 卡；用户明确授权且有参考源时可做可回滚 L-lite。
- 不做破坏性 schema / migration / SSOT 重写。
- 不删除既有文件，除非用户明确点名。
- Git 禁用：`reset --hard` / `push --force` / `rebase` / 删除 commit。
- 允许：启动同步 push base 一次；后续正常 push 到 `auto/YYYY-MM-DD-HHMM`。
- 不引入新依赖，除非项目规范明确且风险可控。
- 不动安全 / 钱 / 权限 / 鉴权核心逻辑。
- 不动 `.env` / 密钥 / 证书 / 生产部署配置。
- 不触发项目文档规范化。
- 单 M 卡上限 30 分钟。
- 单卡审计连续 2 次不通过则跳过并补给下一张。

## 启动序列

1. 跑启动前自检。
2. `git fetch --all --prune`。
3. 记录 base 分支。
4. 若启用 push，启动同步推送 base 分支一次。
5. 落盘本文件到 `docs/施工日志/<今天>/无守护启动-HHMM.md`。
6. `git switch -c auto/YYYY-MM-DD-HHMM`。
7. 启动防睡眠并记录 pid。
8. 落盘队列文件。
9. 启动心跳机制。
10. 进入主循环 Step B。

**用户明确给出无人值守时长、范围、push 策略且无禁止项时，不得再用开放式追问拖延启动。**
