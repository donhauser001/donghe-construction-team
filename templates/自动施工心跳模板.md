# 自动施工心跳

> 自动模式下每 30 分钟追加一行。
> 落盘位置：`docs/施工日志/<今天>/自动施工心跳.md`
> 适用 playbook：`playbooks/16-无守护自动施工模式.md`

---

## 头部（一次性写入）

| 字段 | 值 |
|---|---|
| **启动时间** | YYYY-MM-DD HH:MM |
| **预计结束** | YYYY-MM-DD HH:MM |
| **总时长** | X 小时 |
| **自动施工分支** | `auto/YYYY-MM-DD-HHMM` |
| **队列文件** | `docs/任务卡/_自动施工队列-YYYY-MM-DD-HHMM.md` |
| **caffeinate PID** | xxxxx |
| **持续语义** | 单焦点即停 / 队列跑到时间到 / 队列空后补安全卡 |
| **握手对象** | 无 / X 队 |
| **握手间隔** | N 分钟 |

---

## 心跳行格式（追加式）

每 30 分钟追加一行，格式：

```
[HH:MM] 已用 X:YY / 总 A:BB · 完工 N · 跳过 M · 阻塞 K · 当前 #NNN-<任务名> · 状态 <emoji 状态> · 分支 auto/xxx
```

字段含义：

- `[HH:MM]`：本次心跳时间
- `已用 X:YY`：自启动累计用时
- `总 A:BB`：用户给定的总时长
- `完工 N`：已完工卡数
- `跳过 M`：被跳过卡数（含护栏跳过 / 审计连续不过）
- `阻塞 K`：被阻塞卡数
- `当前 #NNN-<任务名>`：正在做的卡（如无则填"无"）
- `状态 <emoji>`：当前主循环状态（🔨 施工中 / 🔍 待审计 / ⏸️ 阻塞 / 🟢 队列空闲）
- 多队半守护时，心跳行尾必须追加：`· 握手 X队：无冲突 / 有占用需避让 / 已记技术债并避让`

## 异常事件（穿插式追加）

非常态心跳（出现就立刻追加）：

```
[HH:MM] ⚠️ 异常 · <一行描述>
```

示例：

```
[03:15] ⚠️ 异常 · 连续 3 次阻塞，触发熔断，准备退出主循环
[03:16] ⚠️ 异常 · 检测到尝试改 schema，工人已被叫停，本卡 ❌ 跳过
[06:00] ⚠️ 异常 · caffeinate 已自动到期失效，重启失败，主循环可能受电脑睡眠影响
[06:30] ⚠️ 异常 · 队列已空，按持续运行契约执行续跑策略：补下一阶段安全卡
```

```

退出时间：YYYY-MM-DD HH:MM
退出原因：xxx
最终数据：完工 N · 跳过 M · 阻塞 K
退出自证：时间是否已到 / 续跑策略是否已执行 / 握手是否完整
总报告：docs/施工日志/<今天>/自动施工总报告-HHMM.md
==============================
```
