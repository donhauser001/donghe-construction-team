# 任务卡 · YYYY-MM-DD-{队前缀}NNN-任务名称

> **元信息**
> - 编号：YYYY-MM-DD-NNN（单队） / YYYY-MM-DD-{队前缀}NNN（多队）
> - 签发时间：YYYY-MM-DD HH:MM
> - 签发人：包工头
> - **派给**（三元绑定字段 1）：工人#1 / 工人#2 / 工人#3 / **包工头亲自** / **包工头切监理姿态**
> - **调用模型**（三元绑定字段 2）：具体 slug（从 `playbooks/03-派工与并行.md` §三 档位表选）
>   - 包工头亲自时填"父 agent 模型"（如 `claude-4.6-opus-high-thinking`）
> - **身份-模型匹配理由**（三元绑定字段 3）：一句话写明为何这档模型对这档任务"够用即可"（防杀鸡用牛刀 / 防大材小用 / 防默认继承贵模型）
> - 半径：S / M / L
> - 升档触发器（若有）：xxx
> - **所属施工队**：A 队 / B 队 / hotfix 队 / —（单队模式留空）
> - **涉及子项目**：backend / admin-web / miniapp-companion / ...（一卡只能一个子项目）
> - **`parent_focus`**（v0.4 必填 · 八大宪法第 8 条）：`docs/下一阶段工作.md#focus-1`
>   - 散落场景可指 `docs/<散落文档>.md#<锚>`（如 `docs/科科灵2.0-核心业务链路梳理-v1.md#L05`）
>   - 冻结期 S 级豁免活：`冻结期 S 级豁免（无方向锚点）`
>   - 详见 `playbooks/02-任务卡与工人协议.md` §二 + `playbooks/01-接活流程.md` §六
>
> ⚠️ **三元绑定双闸门**：
> - **派工闸**（包工头）：派 Task 前跑 `scripts/check_card.py <本卡路径>` + Task 调用 `model=` 与卡上一致
> - **开工闸**（工人）：接卡首动作核对 prompt 贴的三字段与卡一致 · 不一致 = 退卡（卡顶留备注 + 转回包工头）
