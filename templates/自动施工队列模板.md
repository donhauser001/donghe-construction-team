> 自动模式启动时由包工头规划完整队列后落盘。队列不是一次性清单，而是无人值守循环的状态机。
> 落盘位置：`docs/任务卡/_自动施工队列-YYYY-MM-DD-HHMM.md`
> 适用 playbook：`playbooks/16-无守护自动施工模式.md`

---

## 队列规则

- 优先级数字越小越优先。
- 同优先级按文件域 / 子项目分组。
- 优先做不依赖外部决策的卡。
- 优先做修复类，再做新增类。
- 优先做小卡，再做大卡。
- 用户给了参考项目 / 文档时，优先从参考源映射安全业务卡。
- **队列清空不是退出原因**；时间未到必须进入队列补给。
- 遇到决策问题不停止；按推荐方案继续，重大取舍写入 `docs/技术债账本.md`。
- 用户目标完成但时间未到时，自动从蓝图下放路线图、下一阶段工作、任务卡。

## 队列补给器

| 顺序 | 来源 | 动作 |
|---|---|---|
| 1 | 当前模块 | 补当前模块下一张自然任务卡 |
| 2 | 下一阶段工作 | 补同业务线可回滚卡 |
| 3 | 项目路线图 / AGENTS.md / README | 补明确未完成项 |
| 4 | 用户指定参考项目 / 文档 | 映射可迁移业务能力 |
| 5 | 既有蓝图 / 路线图 | 自动下放下一步方向，标记待用户复核 |
| 6 | QA / 测试 / 类型 / 样式一致性 | 补验证或质量卡 |
| 7 | 缺口清单 | 写技术债账本后继续做审计 / 文档对齐卡 |

补给记录必须写入本文件的"补给记录"段。

## 自动模式护栏自检

| 检查项 | 通过条件 | 不通过的处理 |
|---|---|---|
| 半径 | S / M / 授权 L-lite | 真 L 级 -> 写技术债账本 + 按推荐安全路径继续或补给下一张 |
| 数据 | 不含破坏性 schema / migration | 写技术债账本 + 跳过破坏性动作 + 补给下一张 |
| 配置 | 不含 .env / 证书 / 生产部署 | 跳过 |
| 安全 | 不含安全 / 钱 / 鉴权核心 | 跳过 |
| Git | 不需要 force push / rebase / reset-hard | 停止并报告 |
| 依赖 | 不引入新依赖 | 跳过或待决策 |
