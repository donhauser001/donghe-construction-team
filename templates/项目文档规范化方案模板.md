## 四、等价物映射建议（关键决策）

> 用户已有文档与东合标准角色的对应关系。

| 东合角色 | 用户已有文档 | 处置策略 | 用户确认 |
|---|---|---|---|

| 路线图 | `医疗陪诊平台-开发计划-v1.md` | A ☐ / B ☐ / C ☐ / D ☐ | ☐ |
| 项目特殊文档（API 契约） | `科科灵2.0-P0-API契约-v1.md` | 移动到 `docs/参考资料/` ☐ / 不动仅在 AGENTS.md 映射 ☐ | ☐ |
| ... | ... | ... | ☐ |

**策略说明**：

| 策略 | 含义 | 优 | 缺 | 推荐场景 |
|---|---|---|---|---|
| A · 薄壳指向 | `docs/<角色>.md` 创建薄壳 + 链接到原文档；原文档不动 | 零破坏 + 项目布局规范 | 薄壳与原文档要同步 | **默认推荐** |
| B · 移动归档 | 把原文档 `git mv` 到 `docs/参考资料/`；薄壳指向新位置 | 项目根更整洁 | 可能破坏外部引用 | 项目根杂乱 |
| C · 重命名替代 | 原文档 `git mv` 为 `docs/<角色>.md` | 最干净 | 破坏性大 | 不推荐，需用户明确同意 |
| D · 仅 AGENTS.md 映射 | 完全不动文件，仅 AGENTS.md 写映射 | 完全零破坏 | 项目布局不规范 | 用户拒绝任何文件移动时回退 |

> - 用户可以自己改（推荐）

## 七、不会动的范围（铁律）

> 明确列出**不会触碰**的文件类型。如有疑问 → 停下问用户。

- ❌ 任何 `.ts .tsx .js .jsx .py .go .rs .java` 等代码文件
- ❌ 任何 `package.json / tsconfig.json / Cargo.toml` 等配置
- ❌ 任何 `.sql / migration / schema.prisma` 等数据库文件
- ❌ 任何 `.env / Dockerfile / docker-compose.yml / .gitignore / Makefile`
- ❌ 任何业务数据 JSON / YAML / TOML
- ❌ git 历史（不 reset / 不 rebase / 不 force push / 不删 commit）
- ❌ 删除任何文件（哪怕"明显重复"也只移动，不删）

## 九、等待用户拍板

请按段（二、三、四、五）**逐项确认 / 取消**。也可以一次性说"按推荐执行"。

确认后，包工头：

1. 进入 `docs/施工队-临界占用.md` 注册"规范化进行中"（多队场景）
2. 按确认清单逐项 `git mv`（不批量、可回溯）
3. 按确认清单创建空骨架
4. 按用户选择的策略写 `AGENTS.md` 等价物映射
5. 输出《规范化完成报告》（含已动文件清单 + 受影响引用清单 + 建议 commit 信息）
6. **不自动 git commit**——是否入库由用户决定
7. 释放 `docs/施工队-临界占用.md` 的占用
