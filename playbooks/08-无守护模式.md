# 08 · 无守护模式

> 适用：用户告知要离开，让施工队在给定时间内自主持续施工，回来后可审计、可回滚、可接管。
>
> 核心：**用户给多长时间就按多长时间建契约；宁做小而连续的安全工作，不空跑、不早退、不单卡做完就下班。遇到决策问题不停车，按推荐方案推进，重大取舍记入技术债账本。**

## 一 · 启动

### Step 0 · 确认 4 件必填事（用户已明确给出的不重复问，不得用开放式追问拖延启动）

1. **持续多久**：小时级或明确截止时间。
2. **施工范围**：清空在途卡 / 完成某模块 / 自由发挥 / 参考指定项目推进。
3. **commit/push 策略**：A 自动 commit + 启动同步推 base 一次 + 后续只推 `auto/YYYY-MM-DD-HHMM`（默认）；B 自动 commit 到 auto 分支不 push；C 都不做（用户明确要求才选）。
4. **阻塞策略**：A 跳过继续下一张（默认）；B 选最保守方案继续；C 停下等回（用户明确要求才选）。

### 持续运行契约（启动确认里必须写明）

| 契约项 | 硬规则 |
|---|---|
| 时间窗 | 必须有开始 / 截止；没有就追问，不得自设"单卡结束即止" |
| 队列 | 启动 ≥ 3 张候选卡（4h 以上 ≥ 5 张）；空了必须补给 |
| 续跑 | 单卡完成 / 段落完成 / 队列清空 / 遇到决策点 / 用户目标完成，**都不是退出理由** |
| 参考源 | 用户说"参考 X"→ X 就是决策源，必须主动读并映射任务，不得以业务不清为由停下 |
| 决策 | 不等用户；按推荐方案继续，重大取舍入 `docs/技术债账本.md` |
| 心跳 | 每 30 分钟追加：已用 / 剩余 / 当前卡 / 完工数 / push 状态 |
| 空跑 | 每 10~15 分钟检查实物产出（代码 / 测试 / 文档 / commit / QA）；连续 2 周期无产出 = 流程违规，停止并报告 |
| 合法退出 | 仅：时间到 / 用户回来 / 连续 3 次阻塞熔断 / 护栏违规 / 系统或 Git 不可恢复异常 / base 同步推送失败且用户未允许降级 |
| 退出自证 | 总报告必须答"是否跑满"；未跑满列硬阻塞证据 + 已尝试的补给路径 |

**事故反例**：用户给 4 小时并指明参考项目，包工头 20 分钟后以"队列耗尽 / 等用户决策"退出——重大流程缺陷。

### 启动自检（任一不过拒绝启动）

项目已接入东合；非多队（用户明确要求握手协同除外，届时填握手对象 / 间隔 / 边界并登记临界占用）；git 工作区 clean（或用户授权 preflight 处理）；4 必填 + 契约已确认。然后：fetch 记录 base → 按策略推 base 一次 → 建 `auto/YYYY-MM-DD-HHMM` 分支 → 落盘启动确认 / 队列 / 心跳文件（`templates/无守护施工套件模板.md`）→ 防睡眠（macOS：`caffeinate -i -t <秒> &`，记 pid 退出时 kill；Linux：`systemd-inhibit`；不可用则告知用户风险后继续）。

## 二 · 护栏

**半径**：默认只做 S / M。用户明确授权且给了参考源 / 明确范围时可做 **L-lite**（可回滚、无破坏性的模型补齐 / 管理页搭建 / 测试补齐）。触发真 L 级决策 → 不停工：写技术债账本，按推荐的可回滚路径继续；推荐路径触碰禁区 → 记债切下一张安全卡。

**永远不自动做**：破坏性 schema 迁移；生产数据操作；支付 / 安全 / 鉴权核心逻辑；密钥配置；不可逆删除；依赖体系大改；删除任何既有文件（用户点名除外）；引入新依赖；触发文档规范化（要用户在场）。

**Git 纪律**：施工成果永不落 base / 保护分支（main / master / production / release/*）；每张完工卡单独 commit，message 前缀 `[auto] [card YYYY-MM-DD-NNN]` + 卡链接 + 证据摘要；禁 force push / rebase / reset --hard / 删 commit / 为 push 成功丢弃用户改动。push 失败重试 3 次，auto 分支仍失败 → 继续本地 commit 标记 degraded；base 启动同步失败 → 停（除非用户允许降级）。

**熔断表**：连续 3 次阻塞 → 退出出总报告；单卡审计 2 次不过 → 跳过记禁忌候选；单 M 卡 > 30 分钟 → 切下一张标阻塞；用户发任何消息 → 立刻退出（先收尾再回应）；检测到自身违反护栏 → 立刻退出保留证据。

**模型**：包工头 / 监理强档；工人强制便宜档；心跳 / 文档便宜档。

## 三 · 队列与补给

启动前落盘队列（`docs/任务卡/_自动施工队列-YYYY-MM-DD-HHMM.md`）：5~20 张候选，每张含优先级 / 半径 / 子项目 / 文件域 / 验证方式。排序：无外部依赖优先 > 修复类 > 小卡 > 参考项目可映射的业务卡。

**队列空且时间未到，按序补给**（不许把队列空当自然结束）：

1. 当前模块的下一张自然卡
2. 下一阶段工作的同模块任务
3. 路线图 / README / AGENTS.md 未完成项
4. 用户指定参考源中可迁移的业务能力
5. **蓝图下放器**（用户目标已完成时，见下）
6. 测试 / QA / 样式一致性 / 文档补齐 / 类型修复 / 重复代码收敛
7. 仍无 → 写缺口清单 + 技术债账本，做验证 / 审计 / 文档对齐卡直到时间到或熔断

**蓝图下放器**（方向链确认门槛的无守护例外）：用户目标完成但时间未到 → 读蓝图（或映射等价物）→ 选"最高价值、依赖最少、可回滚"的 💭 条目 → 下放路线图节点 → 下放阶段焦点 → 拆卡（含 parent_focus），每层标"**无守护自动下放，待用户复核**" + 技术债账本留痕。限制：只从已有蓝图 / 路线图 / 参考源取材，**不新造愿景**；蓝图为空 → 退回参考源 / QA / 测试 / 文档债补给，不冻结空等。

**技术债账本**（`docs/技术债账本.md`，用 `templates/技术债账本模板.md`）：每条含时间 / 来源卡 / 类型 / 严重度 / 推荐方案 / 已采取动作 / 待用户确认问题 / **回滚方式** / **关联 commit**。

## 四 · 主循环（示意）

```
deadline = start + 用户时长
while now < deadline:
    用户来消息 / 护栏违规 / 连续 3 阻塞 → break
    队列空 → 补给器；仍无代码卡 → 生成验证/QA/文档卡
    用户目标已完成 → 蓝图下放器 → 结果入队列
    取最高优先卡 → 拆卡落盘 → 触碰禁区？跳过+记债 → 派 1 工人（便宜档）
      → 真实验证 → 审计
      → 过：git add 白名单 → commit [auto][card] → push auto 分支（重试 3）
      → 不过且可重做 < 2 次：重做；否则跳过/阻塞，连续阻塞 +1
    每 10~15 min 空跑检查；每 30 min 心跳
退出：kill caffeinate → 最终 push 一次 → 写总报告 → AGENTS.md 自动施工历史加一行
     → 不切回原分支（auto 分支留给用户审）
```

## 五 · 心跳与总报告

心跳追加到 `docs/施工日志/<今天>/自动施工心跳.md`：

```
[12:30] 已用 0:30 / 总 4:00 · 完工 1 · 跳过 0 · 阻塞 0 · 当前 #003-xxx · 分支 auto/... · push ok
```

总报告必含：起止 / 给定时长 / 实际时长 / **是否跑满** / 退出原因；base 与 auto 分支 push 状态；完工清单（卡 + commit hash + 证据）；跳过 / 阻塞清单（原因 + 已尝试动作）；补给记录；蓝图下放记录；技术债摘要；空跑检查记录；**待用户复核清单**；下一步建议（diff 全貌 → squash merge / cherry-pick / 废弃）；退出自证。

**用户回来的标准回复**：跑了多久（vs 给定）、完工 N 跳过 M 阻塞 K、分支与 push 状态、退出原因、总报告路径、最需要决策的 N 件事、下一步建议。

**旧 auto/ 分支**：用户处置后在 AGENTS.md 历史表标 merged / cherry-picked / discarded；累计未处置 ≥ 3 时接活时提醒清理；包工头不主动删未确认的分支。

## 六 · 反例

没确认时间窗就开跑；给 4 小时跑 20 分钟说队列耗尽；给了参考项目不去参考；遇决策点停下等用户（应记债继续）；目标完成不读蓝图下放；做了不可逆 L 级决策；动破坏性 schema / 删文件 / force push；在 main 上 commit 施工成果；队列空了不补给；心跳没写；用户回来后还继续跑；push 失败不重试不记录；空跑两周期还假装在推进；为跑满时间做无意义改动。
