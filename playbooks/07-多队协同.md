# 07 · 多队协同

> 适用：用户在另一 IDE / 窗口又开一队，共享同一项目根。核心：**默认轻量（单队零开销），按需加重**；多队只在冲突、临界占用、方向 / 生产风险上打扰用户。

## 一 · 模式判定

每次开工自动检测，满足任一即多队模式：`docs/施工队-临界占用.md` 存在且非空；`docs/施工日志/<今天>/` 是目录；任务卡有别队前缀；AGENTS.md"已接入施工队"列表含别队。

**二次主动询问**（防首次切换盲点，per-session 一次）：自动检测 = 单队 + 本 session 首次接活 + 项目接入 ≥ 24h + 用户消息无单/多队信号 → 问一句"是否在其他窗口开了别的队？"。用户没回 → **等**，不默认单队开工。其余场景不问（不烦扰用户）。

## 二 · 硬规则

- **代号**：用户给定，项目内唯一（A 队 / B 队 / hotfix 队）；写入日志开头、卡元信息、AGENTS.md 列表、占用记录。
- **任务卡**：编号空间分队（`A001` / `B001`）；已有无前缀卡视为主队的，不回头改名。**自己的卡自己写，不动别队的卡。**
- **施工日志**：按队分写 `docs/施工日志/YYYY-MM-DD/<队>.md`；切多队当日把已有单文件移为 `A队.md`；总览由主队维护（可选）。
- **踏勘报告只有主队有权改**；别队要重新踏勘 → 日志记一笔建议，主队拍板。

## 三 · 临界占用（打招呼协议）

不能并发改的全局文件：路线图 / 蓝图 / 下一阶段工作 / 知识库 / 禁忌 / 踏勘报告 / AGENTS.md / 核心 schema 文件 / 禁忌里标注"高频冲突"的文件。

改之前在 `docs/施工队-临界占用.md` 追加占用行（时间 / 队 / 文件 / 预估时长），改完**立刻删除**该行 + 日志记一笔。首次进多队模式时从 `templates/施工队-临界占用.md` 复制骨架。

看到别队占用 → **等**，不插队；超预估 2 倍未释放 → 日志记"建议用户介入"，停下涉该文件的工作。

## 四 · 友邻扫描（多队模式每次接活前必做）

1. 读临界占用文件（实时占用）。
2. 列别队在途卡 → 提取白名单 = "友邻占用范围"。
3. 与本次需求范围有交集 → **停下问用户**："两队都要改 X，要协调吗？"
4. 读别队当日日志，了解其动作 / 阻塞 / 风险。

## 五 · 新队加入（不重新踏勘，轻量对齐）

读最新踏勘报告（信任主队）→ 扫多队现状（占用 / 各队在途卡 / 当日日志 / AGENTS 列表）→ 输出《加入对齐报告》（`templates/施工队加入对齐报告模板.md`：本队代号 / 预计范围 / 别队在工范围 / 冲突风险 / 分线建议）→ 用户确认后写入 AGENTS.md 列表 + 建本队当日日志 → 正常接活。

## 六 · 冲突与退队

**已经踩脚**（git 冲突 / 文件被覆盖）：立刻停工 → 卡置 ⏸️ 写明情况 → **不自己 merge** → 用户裁决（谁回滚 / 协商合并）→ 决策过程沉淀禁忌。

**退队**：AGENTS.md 列表标 ⏸️ 暂停 / ❌ 解散 + 时间；在工卡明确处置（继续 / 转交 / 作废）；日志保留。只剩一队自动回退单队模式。

**汇报增项**：多队下每次汇报附本队代号、本轮占用记录、对友邻的提醒（如"我改了 schema，B 队要重新生成 client"）。

## 七 · 反例

单队用多队后缀；多队不写当日日志；改全局文档不打招呼互相覆盖；自己 merge 冲突不告知用户；假装看不见别队在工卡硬上同一文件；占用后忘释放；两队同时重新踏勘。
