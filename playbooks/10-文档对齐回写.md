# 10 · 文档对齐回写

> 适用：每张任务卡完工后；当日工作结束前；项目阶段切换时。
>
> 文档服务施工，不阻塞施工；高风险 / 阶段收口必须补齐，战时可先跑最小可运行闭环，再并行补文档。v0.4.9 起默认执行文档预算和"默认做后汇报"规则。

## 零、战时文档补位原则

文档是施工队的路标和证据柜，不是施工本体。

- 前线正在冲刺时，总工 / 包工头优先补边界、验收、风险、SSOT，不要求施工队等长文档。
- M/L 级任务卡仍必须有，但可以瘦身为"目标 + 白名单 + 黑名单 + 验收证据 + 风险点"。
- 除 schema / 支付资金 / 生产凭证 / 安全隐私 / 医疗 P0-P1 / 跨队冲突 / 蓝图路线图拍板外，不因文档未完善阻塞最小可运行产物。
- 用户链路没跑通时，不用文档替代实测；文档要记录实测现象、阻塞点、下一步动作。
- 阶段收口、上线闸门、工地禁忌、当日施工日志仍必须回写，避免口径漂移。
- 低风险日志、知识库、P2/P3 禁忌候选默认做后汇报，不向用户逐项确认。
- 每个文档事实必须带同步锚点：任务卡链接 / 证据文件 / commit hash / 测试输出 / code reference 至少一种。

### 文档预算（v0.4.9）

| 半径 | 回写规则 |
|---|---|
| S | 施工日志 1 行；不更新路线图 / 知识库，除非用户明确要求 |
| M | 施工日志 + 必要知识库；证据引用机器生成文件 |
| L | 路线图 / 下一阶段 / 知识库按需更新，但先保证产品链路和证据 |
| 多队 / 无守护 | 心跳只追加状态差异；总报告链接证据，不复制长输出 |

文档回写超过预算且产品未跑通时，先停写文档。

## 一、写什么 / 写到哪 / 谁写

| 文档 | 谁触发更新 | 写什么 | 频率 | 多队规则 |
|---|---|---|---|---|

| `docs/工地禁忌.md` | 包工头 / 监理 | P0/P1 必写；P2/P3 可进入后补队列 | 按需追加 | **走临界占用打招呼协议** |
| `docs/路线图.md` | 包工头 | 节点状态变化（✅ 完成 / 🔄 进行 / 🆕 新增） | 节点级触发 | **走临界占用打招呼协议**（一般主队改） |
| `docs/下一阶段工作.md` | 包工头 + 用户 | 阶段聚焦调整 | 阶段切换时 | **走临界占用打招呼协议** |
| `docs/蓝图.md` | 包工头 + 用户共创 | 概念探讨、未来方向、创意 | 用户主动谈论时 | **走临界占用打招呼协议** |
| `docs/工地踏勘报告.md` | 主队包工头 | 全量重新踏勘后覆盖（旧版归档） | 触发器命中时（详见 13） | **只主队有权改** |
| `AGENTS.md` | 包工头 | 项目核心入口、约定速查 | 重大项目结构变化时 | **走临界占用打招呼协议** |

## 二、施工日志（每天必写）

**触发**：当日工作结束前，包工头主动收口。

**位置**：`docs/施工日志/YYYY-MM-DD.md`

**格式**：用 `templates/施工日志模板.md`

**核心字段**：
- 工地状态（当前阶段 / 路线图位置）
- 当日完成的任务卡（链接）
- 当日新发现的项目知识（要不要进项目知识库）
- 当日新发现的工地禁忌（要不要进工地禁忌）
- 未解决问题与明日预告
- 当日触发的工地禁忌（如果有）

**禁止**：
- 跳过当日施工日志（哪怕只做了 S 级 typo 也要记一笔）
- 把多天工作合并写到一天的日志里
- 写空泛流水账（"今天写了一些代码"）
- 复制大段测试输出；只写证据文件链接和关键结论

## 三、项目知识库（持续追加）

**触发**：完成任务卡时，发现以下任一类知识值得沉淀：

- 命名约定 / 目录约定 / 文件结构约定
- 领域术语 / 业务规则
- 已踩过的坑 + 解决方法
- 性能 / 安全 / 体验的"项目级共识"
- "这个文件其实是核心，不能轻易动"
- "这个接口 / schema 决策的历史原因"
- "X 看起来可以并行，其实有 Y 隐性依赖"

**位置**：`docs/项目知识库.md`

**追加格式**：每条用三级标题（### 知识-N：标题），含场景 / 内容 / 来源（任务卡 ID）/ 时间。
**新建格式**：首次创建 `docs/项目知识库.md` 时用 `templates/项目知识库模板.md`。

**禁止**：
- 把"通用编程知识"写进项目知识库（只写**这个项目特有**的）
- 写超过半年还没人引用的知识 → 定期清理或合并

---

## 四、工地禁忌（按需追加）

**触发**：施工 / 审计过程中发现以下任一情况：

- 工人差点 / 已经做了一个明显的临时方案 / 硬编码
- 监理审计发现一个 P0 / P1 反例
- 用户指出"上次那个 X 别再做了"
- 重复出现的同类失误

**位置**：`docs/工地禁忌.md`

**追加格式**：用 `templates/工地禁忌条目模板.md`

**沉淀流程**：详见 `playbooks/12-工地禁忌识别与沉淀.md`

---

## 五、路线图（节点级触发）

**触发**：

- 一个里程碑完成 → 把对应节点标 ✅
- 发现需要新节点 → 追加到"未来节点"区
- 发现节点优先级变化 → 调整顺序，标"调整原因"
- 发现节点应该作废 → 标 ❌ 并注明原因（不要直接删）

**位置**：`docs/路线图.md`
**新建格式**：首次创建用 `templates/路线图模板.md`。

**禁止**：
- 删历史节点（即使作废也要保留 + 标注）
- 在路线图里写细节（细节属于任务卡 / 下一阶段工作）

---

## 六、下一阶段工作（阶段切换时）

**触发**：

- 完成一个阶段 → 用户拍板下一阶段聚焦点 → 包工头更新此文档
- 阶段中途插入紧急工作 → 包工头主动问用户是否调整聚焦
- 计划外发现的工作（不在原阶段范围）→ 评估是插入还是延后

**位置**：`docs/下一阶段工作.md`
**新建格式**：首次创建用 `templates/下一阶段工作模板.md`。

**关键**：这份文档是"包工头每次接活时'01-世界观对齐'必读项"，写得过粗会让后续每张卡都失焦。

---

## 七、蓝图（概念探讨时）

**触发**：

- 用户主动谈"我想未来做 X"
- 包工头在审计中发现"这个改动应该升级为蓝图级方向"
- 完成一个里程碑后回顾"接下来 6 个月想做什么"

**位置**：`docs/蓝图.md`
**条目追加格式**：每条用 `templates/蓝图条目模板.md` 的格式（未确定语气 + 探讨日期 + 待澄清问题）。

**特点**：
- 用未确定语气写（"考虑 / 设想 / 探索"）
- 标注探讨日期和参与人（"2026-05-06 用户 + 包工头"）
- 每条后面留"现状 / 待澄清的问题"
- 蓝图条目升级为"下一阶段工作"或"路线图节点"时要标注转换

---

## 八、AGENTS.md（重大变化时）

**触发**：

- 首次进入项目（**用户确认踏勘报告后**才创建，详见 00 + 13）
- 项目结构有重大变化（新增子项目 / 重要目录改名）
- 项目核心约定变化（命名 / 入口 / 流程）
- 多队场景：施工队加入 / 退出（更新"已接入施工队"列表）
- 等价物映射变化（项目文档规范化后必更新）

**位置**：项目根 `AGENTS.md`

**职责**：
- 含 `<!-- donghe-construction-team: initialized YYYY-MM-DD · vX.Y -->` 标识
- **东合标准角色 → 实际文件等价物映射表**（详见 15）
- **子项目清单**（monorepo）+ 各子项目根目录入口
- **已接入施工队列表**（多队模式）
- 指向所有项目文档（踏勘报告 / 蓝图 / 路线图 / 下一阶段 / 知识库 / 禁忌 / 任务卡 / 施工日志）
- 声明本项目接入了"东合施工队" skill
- 列出项目入口文件 / 核心目录速查

## 九、文档对齐反例

下面是工地禁忌：

- 完工后没写当日施工日志
- 项目知识库写得像 README（重复无沉淀）
- 路线图节点完成了不去标
- 工地禁忌一年没更新过（说明没在记）
- 文档之间互相矛盾（蓝图说 A，路线图说 B）
- 写了文档但没人会去读（写完位置不对、没在 AGENTS.md 索引）
- 多队改全局文档不打招呼，互相覆盖
- 多队同时写当日总览，覆盖彼此
- 改了 schema 但没在工地禁忌沉淀"友邻队需要重新生成 client"
