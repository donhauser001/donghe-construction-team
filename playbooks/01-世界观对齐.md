# 01 · 世界观对齐

> 适用：每次"接活契约"开始前 / 切换到一个新的工作主题时。

## 一、必读清单（按顺序、按存在与否）

每次接活前，按下面顺序扫描一遍。**已存在的必须读**，不存在的跳过：

| # | 文件 | 用途 | 要从中提取什么 |
|---|---|---|---|
| 1 | `docs/工地踏勘报告.md` | **全局快照（入口）** | **第一项必读**。项目当前实际状态、世界观、等价物映射 |
| 2 | `docs/蓝图.md`（或映射的等价物） | 长期方向 / 创意 | 项目想成为什么；本次需求是否在蓝图里 |
| 3 | `docs/路线图.md`（或映射的等价物） | 历史里程碑 + 未来节点 | 当前在哪个节点；本次需求关联哪个未来节点 |
| 4 | `docs/下一阶段工作.md` | 当前阶段聚焦 | 本次需求是否属于当前阶段主线 |
| 5 | `AGENTS.md` | 项目级 agent 指南 + 等价物映射表 | 项目特殊约定 / 重要入口文件 / 已接入施工队列表 |
| 6 | `docs/项目知识库.md` | 跨任务沉淀 | 已经踩过的坑 / 已经定下的约定 |
| 7 | `docs/工地禁忌.md` | 历史反例 | 本次改动是否会重蹈覆辙 |
| 8 | `docs/施工日志/` 最近 3 天 | 工地实时状态 | 上几天做了什么；有没有半成品；有没有阻塞中的卡 |
| 9 | `docs/任务卡/` 进行中的卡（无 `.完工` `.作废` 后缀） | 在途工作 | 是否有相关 / 冲突 / 可复用的卡 |
| 10 | `docs/施工队-临界占用.md`（多队场景） | 实时占用 | 别队正在改哪个全局文档 |

**规则**：

- 必读清单**第一项是工地踏勘报告**（如果存在）——它是其他文档的索引和入口
- 用户已有等价物文档（如 `docs/科科灵2.0-后端与网页端建设蓝图-v1.md`），按 AGENTS.md 的映射表读到
- 大文件（>20KB）允许只读相关章节，但必须**真的读了**，不能"假装读了"

## 二、读完必须能回答

**包工头的世界观对齐自检**：

- [ ] 项目叫什么、解决谁的什么问题？
- [ ] 项目当前阶段（按路线图）是什么？
- [ ] 本次需求落在路线图的哪个节点？
- [ ] 本次需求是否在 `下一阶段工作.md` 主线内？如果不在，是计划外插入还是优先级变更？
- [ ] 本次需求是否触发任何 `工地禁忌.md` 条目？
- [ ] 本次需求是否影响项目知识库里已定的约定（命名 / SSOT / 接口契约）？
- [ ] 是否存在还在 🔨 施工中 / ⏸️ 阻塞 / 🔍 待审计 的相关任务卡？
- [ ] **多队场景**：本次需求范围是否与别队在工范围有交集？

**任意一项答不出来，必须停下来**：

- 答不出 1~3 → 文档缺失 / 没读全 → 回去补读
- 答不出 4~7 → 需要和用户对齐 → 直接问用户
- 答不出 8（多队） → 进入 `playbooks/14-多施工队协同.md` 的"友邻协调"流程

**禁止"边干边对齐"**。

## 三、多队场景下的友邻扫描（情形 B）

如果本队是 B / C / ... 队（不是首队）：

1. 读 `docs/施工队-临界占用.md`
   - 列出别队当前占用的全局文档
   - 检查本次需求是否需要改这些文档 → 如果是，等占用释放或直接停下问用户

2. 读 `docs/施工日志/<今天>/<别队>.md`
   - 看别队今天做了什么、说了什么阻塞、留了什么风险
   - 在自己的接活契约里 cite 相关动作

3. 列别队的在途任务卡（带别队前缀的卡，无完工 / 作废后缀）
   - 检查文件白名单是否与本次需求范围有交集
   - 如有交集 → 写到接活契约的"风险提示"，让用户决策

详见 `playbooks/14-多施工队协同.md`。

## 四、世界观更新机制

读项目文档过程中如果发现：

- 文档之间相互矛盾 → 在踏勘报告或当日施工日志记一笔，**问用户哪份是权威**
- 文档与代码现状不符 → 标注"文档落后于代码"，纳入 `项目知识库` 待办，**强建议重新踏勘**
- 发现新的领域术语 / 新的约定 → 进入施工后，完工时回写到 `项目知识库.md`
- 发现踏勘报告与实际严重漂移 → **直接建议重新踏勘**（详见 `playbooks/13-工地踏勘对齐.md`）

世界观文档**不允许擅自重写**。增量追加可以；结构改写需要先在汇报里和用户确认；多队场景下还要走打招呼协议。

---

## 五、与"接活契约"的衔接

世界观对齐**先于**接活契约。流程是：

```
读 10 类项目文档 →  自检 7~8 问 →  全部能答 →  写接活契约六栏 →  发给用户
                                  ↓
                          答不出 →  停下问用户 / 补读 / 触发重新踏勘
```

世界观对齐通过后，进入 `00-接活契约.md` 的"四、接活契约六栏"。

---

## 六、世界观对齐反例（工地禁忌候选）

- 跳过工地踏勘报告直接读细节文档
- 没读 AGENTS.md 等价物映射表，导致"找不到蓝图"就以为没有
- 多队场景下不扫友邻，直接和别队踩同一个文件
- 文档与代码不一致时假装没看见
- 用过期的踏勘报告做世界观判断
