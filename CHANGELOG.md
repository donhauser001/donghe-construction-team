## v0.4.8 · 2026-05-08（工人不空转 / 审计异步 / 小问题回炉 / 大问题转新卡）

> 触发：donhauser 项目中，用户在 v0.4.7 六槽编制刚落地后立即纠偏 "不要让工人等着，审计干活时工人继续接新卡，可以等到工人闲置时再去修问题，如果问题较大，直接将问题写新卡，按照新任务去执行"。
> 性质：patch 级实战硬化。v0.4.7 把"审计 ≥ 3 暂停派工"作为唯一节流阀，会导致"审计干活时工人空转"。本版补四条一级原则把审计 lane 与施工 lane 解耦，让工人持续产出。

### 新增能力（四条一级原则）

1. **工人不空转**：工人交完一张即领新卡，不等审计回收。
2. **审计异步**：审计与工人施工解耦——审计干活时工人继续接新卡，**不让工人停下来等结论**。
3. **小问题等闲置回炉**：审计发现的小问题（同任务卡内、不升级半径、不动写集边界）→ 进 owner 工人的"待回炉"清单，等对应工人闲置后由文书写补丁卡派回去。
4. **大问题转新卡**：审计发现的大问题（跨任务卡、半径升级、跨写集、动 schema / 资金 / 权限 / 安全 / 生产 / 医疗 P0-P1）→ 由文书直接拆**新任务卡**按新任务派工，不打断当前工人当前在工卡。

判据"小 vs 大"：
- 小：单文件单段落、改动 ≤ 30 行、不动 export / 不改任务卡边界、不升级半径。
- 大：跨文件 / 跨写集 / 改 schema / 动 SSOT / 升半径 / 涉及资金 - 权限 - 安全 - 生产 - 医疗 P0-P1。
- 判不清 = 按大处理（保守路线，避免小补丁卡偷偷越界）。

### 设计原则

**为什么必须有这个机制**：
- AI 凭直觉接活 = 项目走样的最大风险源
- 没有"方向锚点"的活 = 无法回溯到任何业务价值
- 包工头是"对结果负责"，不是"对听话负责" —— 用户要做无方向的活，包工头有责任拒绝

**为什么跨层取材必须用户确认**：
- 任务卡拆分 = 战术决策（自主）
- 切换阶段焦点 = 战略决策（必须用户）
- 切换路线节奏 = 项目级决策（必须用户）
- 扩展蓝图概念 = 方向级决策（必须对话）

**为什么有"冻结"机制**：
- 防止包工头在无方向状态下"为了干活而干活"
- 让用户意识到"项目已经没有清晰下一步"的事实
- 强迫用户做战略思考 / 项目复盘
- **冻结不是失败，是健康项目的自然节点**（如 MVP 完成后停下问"下一步往哪走"）

### 与 v0.3.2 的关系（v0.3.2 → v0.4 升级路径）

v0.3.2 的"阶段焦点告警 A/B/C/D"是**简化版**，v0.4 把它升级为**完整的四层探查 + 闭环机制**。逻辑保留向后兼容：

- v0.3.2 的 A 情形（找不到下一阶段焦点） → v0.4 情形 D（下一阶段层耗尽）
- v0.3.2 的 B 情形（已完结） → v0.4 情形 D + 阶段切换路径
- v0.3.2 的 C 情形（焦点未拆解） → v0.4 情形 D（路线图有候选但下一阶段无独立文件）
- v0.3.2 的 D 情形（散落） → v0.4 情形 E（保留）
- **v0.4 新增**：A 全空（项目无定义）/ B 蓝图层耗尽（冻结）/ C 路线图层耗尽 / F 健康
